ES6 Set 和 Map 数据结构

Set

ES6提供了新的数据结构Set。类似于数组，只不过其成员都是唯一的，没有重复的值； （set本身是一个构造函数，用来生成Set数据结构。）

Set结构接收一个数组作为参数 ，用来初始化。

注意：向Set加入值的时候。不会发生类型转换。 这意味着，在Set中 5 和 “5” 是不同的值；

Set结构有以下属性：

  1. Set.prototype.constructor :构造函数，默认就是Set函数。
  2. set.prototype.size ： 返回Set的成员总数。

Set结构有以下方法

  • add（value) ：添加某个值。
  • delete（value) :删除某个值。
  • has（value) ： 返回以一个布尔值，表示该值是否为Set的成员。

  • clear（value) ：清除所有成员。

Map

js对象本质上是键值对的集合。但是只能使用字符串充当键。这在一定程度上给我们的使用带来了很大的限制。

而ES6正是为了解决这个问题才提供的Map结构。它类似于对象，也是键值对集合，但是键的范围不不限于字符串，对象也可以当做键。

Map的属性和方法

  • size：返回成员总数。
  • set（key，value）：设置一个键值对。
  • get（key）：读取一个键。
  • has（key）：返回一个布尔值，表示某个键是否在Map结构中。
  • delete（key）：删除某个键。

  • clear()：清楚所有成员。

    遍历
    Map原生提供三个遍历器。

  1. key() : 返回键名的遍历器。
  2. values() : 返回键值的遍历器。

  3. entries() : 返回所有成员的遍历器。

vue中watch的理解

Posted on 2018-12-03

首先确认 watch 是一个对象，一定要当成对象来使用，对象就有键，有值；

键：就是你要监听的那个值，比如说$route,这个就是要监听路由的变化。或者说是data中的某个变量；

值：

1.值可以是函数：当你监听的值发生变化时，需要执行的函数，这个函数有两个形参，第一个是当前值，第二个值是变化后的值。

2.值也可以是函数名：不过这个函数名要用单引号来包裹；

3.值可以是包括选项的对象：（handler，deep，immediate）

  1. 第一个handler：其值是一个回调函数，即监听到变化时应该执行的函数；
  2. 第二个是deep：其值是true或false；确认是否深度监听。（一般监听时是不能监听到对象属性值的变化的，数组的值变化可以听到）

  3. 第三个是 immediate：其值是true或false；确认是否以当前的初始值执行handler的函数；

    export default {
name: "homePage",
data(){
return{
 a:1,
 b:2,
 c:3
}
},
watch:{
a:function (val,oldVal) {
  console.log("a当前值"+ val ,"a变化后的值"+ oldVal );
},
//方法名
b:'changeB',

//选项的对象
c: {
 handler: function (val, oldVal) {
    console.log("c变化了")
 },
 deep: true,
 immediate: true
}
},
created(){
this.a = this.a + 1;
this.b = this.b + 2;
},
methods:{
changeB:function (val,oldVal) {
console.log("当前值"+ val ,"变化后的值"+ oldVal );
}
}
}

  • 划分域名，各域名下的存储空间由各业务组统一规划使用
  • 跨页面传数据，考虑单页应用
  • 清除不需要缓存

三者异同

数据的生命周期：
   cookie：一般由服务器生成，可设置失效时间，如果在浏览器端生成cookie，默认是关闭浏览器后失效
   localStorage: 除非被清除，永久有效
   sessionStorage：仅在当前对话下有效，关闭页面后，被清除

 存储数据大小：
   cookie：4k左右
   localStorage：5MB
   sessionStorage：5MB

 与服务端通信：
   cookie：每次都会携带在http头中，如果使用cookie存储过多数据带来性能问题
   localStorage：仅在客户端（浏览器）中保存
   sessionStorage：不参与和服务器的通信

  易用性：
   cookie：需要自己封装，原生的cookie接口不友好

echart样式系列

范例1：项目中使用echart时会经常要修改图例样式,或者再图例处配置数据,如下图：

首先，小圆点是怎么来得呢？默认是矩形…
其实很简单，我们只需要在设置图例得data里设置icon为pin即可，这就是一个小圆点：

 legend: {
    orient: 'vertical',
    top:'middle',
    right:20,
    itemWidth:16,
    itemHeight:1,
    data:[
        {name:'0~12岁(儿童)  '+'23%',icon:'pin'},
        {name:'13~18岁(少年)  '+'27%',icon:'pin'},
        {name:'18~39岁(青年)  '+'14%',icon:'pin'},
        {name:'40~60岁(中年)  '+'20%',icon:'pin'},
        {name:'61岁以上(老年)  '+'16%',icon:'pin'},
    ]
},

❤ 注意点： 

1. icon分别有['circle', 'rect', 'roundRect', 'triangle', 'diamond', 'pin', 'arrow'];  
2. 设置itemWidth设置想要的大小，注意这里 itemHeight设置为与itemWidth相等或者为1才会是⚪；    
3. 图例中的data的name值要与实际的data的name值一致，数据可以用一个变量拼接上去即可；

VUEX

每一个 Vuex 应用的核心就是 store（仓库）。

“store”基本上就是一个容器，它包含着你的应用中大部分的状态 (state)。Vuex 和单纯的全局对象有以下两点不同

  • Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。

  • 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。
