echart样式

Posted on 2018-06-07

echart样式系列

范例1：项目中使用echart时会经常要修改图例样式,或者再图例处配置数据,如下图：

首先，小圆点是怎么来得呢？默认是矩形…
其实很简单，我们只需要在设置图例得data里设置icon为pin即可，这就是一个小圆点：

 legend: {
    orient: 'vertical',
    top:'middle',
    right:20,
    itemWidth:16,
    itemHeight:1,
    data:[
        {name:'0~12岁(儿童)  '+'23%',icon:'pin'},
        {name:'13~18岁(少年)  '+'27%',icon:'pin'},
        {name:'18~39岁(青年)  '+'14%',icon:'pin'},
        {name:'40~60岁(中年)  '+'20%',icon:'pin'},
        {name:'61岁以上(老年)  '+'16%',icon:'pin'},
    ]
},

❤ 注意点： 

1. icon分别有['circle', 'rect', 'roundRect', 'triangle', 'diamond', 'pin', 'arrow'];  
2. 设置itemWidth设置想要的大小，注意这里 itemHeight设置为与itemWidth相等或者为1才会是⚪；    
3. 图例中的data的name值要与实际的data的name值一致，数据可以用一个变量拼接上去即可；

Posted on 2017-10-12

VUEX

每一个 Vuex 应用的核心就是 store（仓库）。

“store”基本上就是一个容器，它包含着你的应用中大部分的状态 (state)。Vuex 和单纯的全局对象有以下两点不同

  • Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。

  • 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。

webStorm配置less以及压缩css

Posted on 2017-05-12

websrorm下配置less环境 and 设置压缩css

首先,webstorm这个强大的编辑器是自带less的，不过编译过程是需要nodejs环境。

步骤

  1. 下载对应版本的NODEJS： HTTPS://NODEJS.ORG/。

    1. 安装完成后打开命令符(WIN+R）,分别输入NODE-V以及NPM-V，如果返回版本号说明你已经安装成功了。如下图：
    2. 安装Less：npm install less。
    3. 配置webstorm：file ->settings ->Tools ->File Watchers,点击右上角的“加号”标志add，进入对话框，如下图：
    5. program里设置的是：lessc编译路径，它会默认编译到根目录。例如：/Users/xx/node_modules/less/bin/lessc。
    6. 如果你想顺便压缩编译后的css，除了在Arguments中写入–plugin=less-plugin-clean-css外还要安装压缩插件：
      npm install -g less-plugin-clean-css，不过这里需要注意的是：less-plugin-clean-css的路径和lessc编译路径的/Users/xx/node_modules/less应一致；
    7. 你可以为你的编译器指定一个快捷键，然后你可以为这个编译器指定一个快捷键，我使用的是ALT + B；也可以手动右键css ->less；如下图：
    9. 之后打开一个.less后缀名的文件，快捷键编译，如果webstorm底部状态控制器显示 Process finished with exit code 0 ，那么恭喜你，大功告成！

    小问题

  1. 明明已经安装nodejs了，但总提醒你nodejs不存在，这个时候可以看一下你的nodejs安装的路径。

  1. 出现乱码提示的问题，说明nodejs没有安装成功或者安装有问题，解决办法忒粗鲁，卸载重新安装nodejs，安装好后切记重启电脑；
